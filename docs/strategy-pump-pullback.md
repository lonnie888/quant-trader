# Pump Pullback 策略说明 v1.1

> 策略目标：识别币安永续合约的"泵→回撤→二段起跳"形态，在第二波启动时入场。
>
> v1.1 参数基于 17 个涨幅榜币种 × 1 个月 15m K 线数据的网格回测优化。

---

## 策略逻辑（7道关卡）

```
第1关 ── 泵检测
  回看最近 12 根 15m K线。
  若 (最高价÷最低价 - 1) ≥ 10%，标记为"泵"，记录泵的峰值价和峰值量。

第2关 ── 泵未过期
  最近的泵在 24小时(96 bar) 内，否则忽略。

第3关 ── 回撤在范围内【核心】
  当前价格从泵峰值回调了多少？
  要求: 5% ≤ 回撤 ≤ 30%

第4关 ── 回调缩量
  最近4根K线均量 ≤ 泵峰值量的 80%
  → 原理: 回调时没人买，量越小说明抛压衰竭

第5关 ── EMA趋势确认
  收盘价 ≥ EMA(12)（在12周期均线上方）
  → 确保不是下跌趋势中抄底（无额外硬门槛）

第6关 ── 二次放量
  （已禁用：vol_recover=1.0，无额外放量要求）
  → 回测表明放量要求过滤掉过多有效信号

第7关 ── 冷却期
  距离上一次平仓 ≥ 24 根K线(6小时)
  → 防止一个币连续开
```

## 参数详解

### pullback_min / pullback_max — 回撤范围

```
泵峰值价 = 1.00
当前价 = 0.92
回撤 = (1.00 - 0.92) / 1.00 = 8%

条件: 5% ≤ 8% ≤ 30% → ✅ 允许入场
```

- **pullback_min=5%**：必须至少回调 5% 才考虑入场，过滤掉不回撤就追入的假信号
- **pullback_max=30%**：回调超过 30% 说明趋势已反转，不再入场

### vol_shrink — 缩量系数

```
泵峰值量 = 100 万
vol_shrink = 0.80

条件: 最近4根K线均量 ≤ 100万 × 0.80 = 80万 → ✅
```

回调时成交量必须缩小到泵峰值量的 80% 以下。缩量越狠，说明抛压衰竭越充分。

### vol_recover — 放量系数

```
回调均量(4根) = 50 万
vol_recover = 1.0

条件: 最近7根K线均量 ≥ 50万 × 1.0 = 50万 → ✅ (总是通过)
```

回测发现 `vol_recover=1.0`（去掉二次放量要求）收益最高。设为 1.0 意味着只要量能不继续萎缩就行，不需要显著放量。

### trigger_pct — EMA 触发阈值

```
EMA(12) = 1.00
trigger_pct = 0.0

条件: 收盘价 ≥ 1.00 × 1.0 = 1.00 → ✅ (只要在EMA上方)
```

设为 0 表示只要价格在均线上方即可入场，不需要额外高出一定比例。回测证明去掉这个硬门槛收益更高。

### ema_period — EMA 周期

```
ema_period = 12 ≈ 3小时的均线
ema_period = 9  ≈ 2.25小时的均线
```

周期越大，均线越平滑，对短期波动的反应越迟钝。12 比 9 更稳定，过滤掉更多假信号。

### hold_bars — 持仓周期

```
hold_bars = 48 ≈ 12小时

入场后最多持有 48 根 15m K 线（12小时），到期未触发止损则强制平仓。
```

回测表明 48 根（12小时）比 24 根（6小时）收益更高，因为泵→回撤→二段起跳的完整周期通常需要更长时间。

### stop_loss_pct — 止损比例

```
入场价 = 1.00
stop_loss_pct = 5%

止损价 = 1.00 × (1 - 0.05) = 0.95
```

5% 止损比 10% 更紧，回测显示收益更高（+803% vs +620%），因为及时止损避免了深度回撤。

### cooldown — 冷却期

```
平仓后等待 24 根 K 线(6小时)才能再次开同一个币。
```

避免一个币连续反复开仓。24 根比 12 根效果更好，给足时间形成新的泵形态。

## 回测结果

### 测试环境
- **数据范围**: 2026-06-10 ~ 2026-07-08（1个月 15m K线）
- **测试币种**: 17 个真实涨幅榜币种
- **杠杆**: 3x
- **评分公式**: `Score = 总收益 × (1 + 胜率) - 最大回撤 × 0.3`
