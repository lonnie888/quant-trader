# 开仓流程说明

> 每 15 分钟运行一次，扫描涨幅榜前 30 名，检查是否有符合条件的开仓信号。

---

## 完整流程

```
┌─────────────────────────────────────────────────────────────┐
│ 1. 拉涨幅榜 Top 30                                         │
│    GET /fapi/v1/ticker/24hr                                 │
│   筛选条件: USDT交易对, 24h成交量 ≥ 2000万U                  │
│   输出: 涨幅前30的币种列表 (ccxt格式: XXX/USDT:USDT)         │
└──────────────────────────┬──────────────────────────────────┘
                           ↓
┌──────────────────────────┴──────────────────────────────────┐
│ 2. 遍历每个币                                               │
└──────────────────────────┬──────────────────────────────────┘
                           ↓
              ┌────────────┴────────────┐
              │ 冷却期检查              │
              │ 1h内timeout退出的币?     │
              └────────────┬────────────┘
              ✅ 否         │         ❌ 是 → 跳过
                           ↓
              ┌────────────┴────────────┐
              │ 已有持仓检查            │
              │ 该币已有open持仓?        │
              └────────────┬────────────┘
              ✅ 否         │         ❌ 是 → 跳过
                           ↓
              ┌─────────────────────────┐
              │ 3. 缓存数据检查          │
              │    (ParquetStore)       │
              └────────────┬────────────┘
              ┌────────────┴────────────┐
              │ 数据够新吗?              │
              │ 最后1根K线距现在≤3h      │
              │ 且≥100根                │
              └────────────┬────────────┘
              ❌ 旧/不足    │         ✅ 够新
                           ↓
              ┌─────────────────────────┐
              │ 拉7天15m K线             │
              │ GET /fapi/v1/klines      │
              │ 参数: symbol, interval=15m│
              │  limit=1000             │
              │ 保存到ParquetStore        │
              │ 输出: 672根K线(7天)       │
              └────────────┬────────────┘
              ┌────────────┴────────────┐
              │ 数据≥100根?              │
              └────────────┬────────────┘
              ✅ 是         │         ❌ 否 → 跳过
                           ↓
              ┌─────────────────────────┐
              │ 4. 跑 pump_pullback 策略  │
              │    (v1.2参数)            │
              └────────────┬────────────┘
              ┌────────────┴────────────┐
              │ 第1关: 泵检测            │
              │ 回看最近12根15m K线      │
              │ (最高价÷最低价-1) ≥ 13%  │
              │ → 记录泵峰值价和峰值量   │
              └────────────┬────────────┘
              ✅ 通过       │         ❌ 不通过 → 跳过
                           ↓
              ┌────────────┴────────────┐
              │ 第2关: 泵未过期          │
              │ 最近的泵在24h(96根)内    │
              └────────────┬────────────┘
              ✅ 通过       │         ❌ 不通过 → 跳过
                           ↓
              ┌────────────┴────────────┐
              │ 第3关: 回撤在范围内      │
              │ 当前价从泵峰值回调了     │
              │ 5% ≤ 回撤 ≤ 30%        │
              └────────────┬────────────┘
              ✅ 通过       │         ❌ 不通过 → 跳过
                           ↓
              ┌────────────┴────────────┐
              │ 第4关: 回调缩量          │
              │ 最近4根K线均量           │
              │ ≤ 泵峰值量 × 0.80      │
              │ (抛压衰竭)              │
              └────────────┬────────────┘
              ✅ 通过       │         ❌ 不通过 → 跳过
                           ↓
              ┌────────────┴────────────┐
              │ 第5关: EMA趋势确认       │
              │ 收盘价 ≥ EMA(12)        │
              │ (trigger_pct=0, 跳过)   │
              └────────────┬────────────┘
              ✅ 通过       │         ❌ 不通过 → 跳过
                           ↓
              ┌────────────┴────────────┐
              │ 第6关: 二次放量          │
              │ (vol_recover=1.0, 跳过) │
              └────────────┬────────────┘
              ✅ 通过       │         ❌ 不通过 → 跳过
                           ↓
              ┌────────────┴────────────┐
              │ 第7关: 冷却期            │
              │ 距上次平仓 ≥ 12根(3h)   │
              └────────────┬────────────┘
              ✅ 通过       │         ❌ 不通过 → 跳过
                           ↓
              ┌─────────────────────────┐
              │ 5. 最后一根K线有信号      │
              │    s[-1] == 1           │
              │    且最近一次0→1 ≥ 0    │
              └────────────┬────────────┘
              ✅ 有信号      │         ❌ 无信号 → 跳过
                           ↓
              ┌─────────────────────────┐
              │ 6. 拉实时价格            │
              │    GET /fapi/v1/ticker/price│
              │    ?symbol=XXXUSDT      │
              │    → 实时市价开单        │
              │    失败则fallback到      │
              │    缓存K线收盘价         │
              └────────────┬────────────┘
                           ↓
              ┌─────────────────────────┐
              │ 7. 风控闸门              │
              │    evaluate_risk()      │
              └────────────┬────────────┘
              ┌────────────┴────────────┐
              │ max_concurrent ≤ 3?     │
              │ 当前持仓数 < 3?         │
              └────────────┬────────────┘
              ✅ 是         │         ❌ 否 → blocked
                           ↓
              ┌────────────┴────────────┐
              │ daily_loss_limit?       │
              │ 今日已实现亏损 ≤ 30%?   │
              └────────────┬────────────┘
              ✅ 是         │         ❌ 否 → blocked
                           ↓
              ┌────────────┴────────────┐
              │ max_total_exposure?     │
              │ 总持仓 ≤ 30%?          │
              └────────────┬────────────┘
              ✅ 是         │         ❌ 否 → blocked
                           ↓
              ┌─────────────────────────┐
              │ 8. 开单                  │
              │    open_position()      │
              │    → 写入positions.jsonl │
              │    → 飞书通知            │
              └─────────────────────────┘
```

## 参数中文对照

| 参数 | 值 | 中文 | 说明 |
|:----|:--:|:----|:-----|
| `pump_window` | 12 | 泵检测窗口 | 回看12根15m K线(3h)检测泵 |
| `pump_threshold` | 13% | 泵涨幅门槛 | 窗口内涨幅≥13%才算泵 |
| `pullback_min` | 5% | 最小回撤 | 从泵峰值至少回调5%才入场 |
| `pullback_max` | 30% | 最大回撤 | 回调超过30%放弃(回不来了) |
| `vol_shrink` | 0.80 | 缩量系数 | 回调均量 ≤ 泵峰值量×0.80 |
| `vol_recover` | 1.0 | 放量系数 | 已跳过(无条件通过) |
| `trigger_pct` | 0% | EMA触发阈值 | 已跳过(只要在EMA上方即可) |
| `ema_period` | 12 | EMA周期 | 趋势确认均线周期 |
| `hold_bars` | 24(6h) | 持仓周期 | 最多持24根K线后强制平仓 |
| `cooldown` | 12(3h) | 冷却期 | 平仓后等12根K线才能再开 |
| `stop_loss_pct` | 10% | 止损比例 | 入场价下方10%止损 |
| `take_profit_pct` | 0% | 止盈比例 | 不止盈，只靠时间退出 |

## 风控参数

| 参数 | 值 | 中文 | 说明 |
|:----|:--:|:----|:-----|
| `max_concurrent` | 3 | 最大并发持仓 | 同时最多持有3个仓位 |
| `max_position_pct` | 10% | 单仓占比 | 单仓≤账户10% |
| `max_total_exposure` | 30% | 总持仓占比 | 总持仓≤账户30% |
| `daily_loss_limit` | 30% | 每日亏损上限 | 杠杆后亏损≥30%当天停止开仓(约1单SL) |
| `leverage` | 3x | 杠杆 | 所有仓位3倍杠杆 |

## 各模块职责

| 模块 | 频率 | 职责 |
|:----|:----:|:-----|
| `_refresh_watchlist` | 15min | 拉涨幅榜→拉K线→跑策略→开仓→飞书 |
| `_rest_poll_loop` | 15s | 实时ticker价格→SL/TP检测→平仓→飞书 |
| `_positions_report_loop` | K线收盘后 | 持仓检查飞书卡片 |
| `_daily_recap_loop` | 每天08:00(北京) | 每日复盘 |
| `sltp_watch.on_mark` | ~15s | SL/TP判定+close_position+飞书通知 |
